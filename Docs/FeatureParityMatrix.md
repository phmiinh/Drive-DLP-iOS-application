# Feature Parity Matrix

| ID | Feature | Local Android Paths | User Flow | Swift iOS Implementation | Status | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| FP-001 | App bootstrap | `cells-client/src/main/java/com/pydio/android/cells/MainActivity.kt` | Launch app, route to login or content | `AppCoordinator`, `AppBootstrapService` | Done | Restores active session when possible |
| FP-002 | Server detection | `sdk-java/src/main/java/com/pydio/cells/transport/ServerFactory.java` | Validate URL, detect Cells vs P8 | `PydioAPIClient.inspectServer` | Done | Uses local-first semantics |
| FP-003 | Cells OAuth login | `AuthService.kt`, `LoginVM.kt`, `CellsTransport.java` | Open browser, callback, token exchange | `AuthSessionService.authenticateCells` | Done | Keeps `cells-mobile` and `cellsauth://callback` |
| FP-004 | Session restore | `AccountService.kt`, `ConnectionService.kt`, `P8Transport.java` | Reopen foreground account | `AppBootstrapService.initialPhase` | Done | Cells refreshes OAuth tokens when possible; legacy P8 revalidates through stored credentials |
| FP-005 | Account list | `ui/account/*` | Show sessions, open, logout, forget | `AccountsView`, `AccountsViewModel` | Done | Local persistence backed |
| FP-006 | Workspace list | `CellsClient.workspaceList`, `P8Client.workspaceList`, `BrowseNavGraph.kt` | Open account root, list workspaces | `BrowseHomeView`, `BrowseRootViewModel.load`, `LegacyP8Client.fetchWorkspaces` | Done | Cells uses `/a/workspace`, legacy P8 uses the XML registry path reconstructed from `P8Transport` |
| FP-007 | Folder browse | `FolderVM.kt`, `CellsClient.ls` | Open folder, paginate children | `FolderBrowserViewModel.load` | Done | Pagination foundation present, UI is first page |
| FP-008 | Search | `SearchVM.kt`, `CellsClient.search` | Search in current folder scope | `FolderBrowserViewModel.search` | Done | Scoped search only in this iteration |
| FP-009 | Create folder | `CellsClient.mkdir`, `P8Client.mkdir` | Create folder in current directory | `FolderBrowserViewModel.createFolder`, `LegacyP8Client.createFolder` | Done | Cells and legacy P8 both wired |
| FP-010 | Rename | `CellsClient.rename`, `P8Client.rename` | Rename node in current folder | `FolderBrowserViewModel.rename`, `LegacyP8Client.rename` | Done | Cells and legacy P8 both wired |
| FP-011 | Delete | `CellsClient.delete`, `P8Client.delete` | Delete node | `FolderBrowserViewModel.delete`, `LegacyP8Client.delete` | Done | Cells and legacy P8 both wired |
| FP-011A | Move and copy | `NodeService.kt`, `NodeActionsVM.kt`, `CellsClient.copy`, `CellsClient.move`, `P8Client.copy`, `P8Client.move` | Pick a destination folder and relocate a node | `FolderBrowserViewModel.relocate`, `RemoteFolderPickerSheet`, `NodeRepository.move`, `NodeRepository.copy`, `LegacyP8Client.relocate` | Done | Current UI covers single-node move/copy with same-workspace semantics that match the Android client contract |
| FP-012 | Transfers queue persistence | `TransferService.kt`, `RTransfer.kt` | Track queued work | `TransferQueueService`, `TransfersView` | Done | Queue persistence, deduplication, cancel, retry, cache retention, and limited concurrency are implemented |
| FP-013 | Upload | `TransferService.kt`, `S3TransferService.kt`, `P8TransferService.kt` | Share/import then upload | `FolderBrowserView`, `TransferQueueService`, `CellsS3Presigner`, `LegacyP8Client.upload` | Partial | Cells direct PUT and legacy P8 multipart upload are implemented, but background sessions and resumable parity are still missing |
| FP-014 | Download and preview | `TransferService.kt`, `DownloadVM.kt`, `P8TransferService.kt` | Open remote file locally | `TransferQueueService`, `LegacyP8Client.download`, `RemoteNodeDetailSheet`, `QuickLookPreviewSheet`, `ShareSheet` | Partial | Cells and legacy P8 downloads both land in local cache with preview/share, but background resume and richer preview parity are still missing |
| FP-015 | Settings and cache cleanup | `Settings.kt`, `HouseKeeping.kt` | Change policies and clear cache | `SettingsViewModel`, `SettingsView` | Done | Clears cached node listings, finished queue entries, and downloaded file cache |
| FP-016 | Legacy P8 support | `P8Transport.java`, `P8Server.java`, `P8Client.java` | Detect, authenticate, browse, transfer | `LegacyP8Client`, `AuthSessionService.loginLegacy`, `NodeRepository`, `TransferQueueService` | Partial | Login, restore, browse, search, move/copy, bookmarks, public-link creation/removal and transfer now run; captcha and some advanced legacy edge cases are still open |
| FP-017 | Logs and jobs | `ui/system/screens/Logs.kt`, `Jobs.kt` | Inspect runtime tasks | `LogsView`, `JobsView`, `LogRepository`, `JobRepository` | Done | Runtime jobs/logs screens are present and fed by transfer + offline sync services |
| FP-018 | Offline roots and sync | `OfflineVM.kt`, `OfflineSyncWorker.kt` | Pin offline content and sync | `OfflineRepository`, `OfflineRootsView`, `OfflineSyncService` | Partial | Pinning and recursive sync preparation are implemented, but the full Android worker/diff engine is still missing |
| FP-019 | Bookmark and public link actions | `NodeService.toggleBookmark`, `NodeService.removeShare`, `CellsClient.bookmark`, `CellsClient.share`, `P8Client.bookmark`, `P8Client.share` | Bookmark a node, create/load/remove a public link | `FolderBrowserViewModel.toggleBookmark`, `FolderBrowserViewModel.sharePublicLink`, `NodeRepository.setBookmarked`, `NodeRepository.createOrFetchPublicLink`, `PydioAPIClient`, `LegacyP8Client` | Partial | Actions now work from the browser and file detail sheet for Cells and legacy P8, but richer share management UI and advanced edge cases are still missing |
| FP-020 | Bookmark listing screen | `BookmarksVM.kt`, `Bookmarks.kt`, `CellsClient.getBookmarks`, `P8Client.getBookmarks` | Refresh and browse all bookmarks for the active account | `BookmarksView`, `BookmarksViewModel`, `NodeRepository.listBookmarkedNodes`, `PydioAPIClient.fetchBookmarkedNodes`, `LegacyP8Client.fetchBookmarkedNodes` | Done | Cells bookmark listing respects `AppearsIn` workspace mappings, and legacy P8 bookmark listing scans workspaces with the same `search_by_keyword` semantics as Android |
